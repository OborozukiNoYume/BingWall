# BingWall 阶段 TODO 路线图

## 文档元信息

- 更新时间：2026-03-23T00:00:00Z
- 依据文档：`docs/system-design.md`
- 文档定位：按阶段拆解的 TODO、依赖关系与验收标准

## 使用说明

- 本文件以系统设计总纲为唯一上位依据
- 每条 TODO 都尽量控制在单一交付目标内
- 验收标准必须能判断“是否完成”，不能只写方向
- 依赖关系仅记录直接前置项，避免范围蔓延

## 阶段一：基础闭环

### TODO-1：初始化后端工程骨架

- 目标：建立一期后端基础目录、配置入口、依赖管理和基础应用启动能力
- 主要产出：后端项目目录、配置加载、依赖清单、基础应用入口、代码格式化和测试框架
- 依赖关系：无
- 验收标准：
  - 仓库中存在可运行的后端工程骨架
  - Python 运行时已固定为单一精确版本，并写入项目管理文件
  - 配置项通过统一入口加载，敏感值不写死
  - 本地可以启动最小 FastAPI 服务
  - 格式化、基础测试命令可执行

### TODO-2：实现数据库模型与迁移基线

- 目标：把一期核心实体落成真实数据表和迁移脚本
- 主要产出：`wallpapers`、`image_resources`、`collection_tasks`、`collection_task_items`、`admin_users`、`audit_logs`
- 依赖关系：依赖 `TODO-1`
- 验收标准：
  - 所有核心表都通过迁移脚本创建，不手工改库
  - 状态字段、唯一约束和关键索引已落地
  - 时间字段统一按 UTC 存储
  - 可在空库完成初始化

### TODO-3：实现 Bing 采集与去重主链路

- 目标：完成从上游获取数据、规范化、去重、写入任务和内容记录的流程
- 主要产出：采集器、字段映射、去重逻辑、任务记录、明细日志
- 依赖关系：依赖 `TODO-2`
- 验收标准：
  - 能成功拉取至少一个市场的 Bing 数据
  - 重复执行不会重复创建同一内容
  - 任务记录能区分成功、重复和失败数量
  - 上游异常会留下可读错误摘要

### TODO-4：实现本地资源下载与文件管理

- 目标：完成图片下载、校验、正式入库和资源状态管理
- 主要产出：临时目录处理、正式目录写入、资源记录、失败隔离能力
- 依赖关系：依赖 `TODO-3`
- 验收标准：
  - 图片先写入临时目录，再转入正式目录
  - 下载成功后资源状态变为 `ready`
  - 下载失败后资源状态变为 `failed`
  - 正式资源目录可被静态服务访问

### TODO-5：实现公开 API 最小集

- 目标：提供公开列表、详情、筛选和站点基础信息接口
- 主要产出：公开 API 路由、查询逻辑、统一响应结构
- 依赖关系：依赖 `TODO-2`、`TODO-4`
- 验收标准：
  - 列表接口只返回 `enabled` 且资源 `ready` 的内容
  - 详情接口可返回完整展示字段和下载地址
  - 参数非法时返回正确 HTTP 状态码和业务错误码
  - 列表接口支持分页

### TODO-6：实现基础公开前端

- 目标：提供首页、列表页和详情页的最小可用页面
- 主要产出：公开站点页面、筛选交互、详情展示、错误页
- 依赖关系：依赖 `TODO-5`
- 验收标准：
  - 访客可查看最新壁纸列表
  - 可进入详情页查看单张壁纸信息
  - 空结果和不存在内容有明确提示
  - 下载入口可访问可用资源

### TODO-7：完成单机基础部署闭环

- 目标：让 Nginx、FastAPI、SQLite、本地文件和 systemd 能在单机上协同工作
- 主要产出：Nginx 配置、systemd 服务、基础部署文档、启动命令
- 依赖关系：依赖 `TODO-5`、`TODO-6`
- 验收标准：
  - API 请求可经 Nginx 转发到应用
  - 图片资源可由 Nginx 直接提供访问
  - 服务可由 systemd 启动和重启
  - README 中存在可复制的启动说明

## 阶段二：后台管理与运维补齐

### TODO-8：实现管理员认证

- 目标：提供后台登录、会话管理和基础安全控制
- 主要产出：管理员账号模型使用、登录接口、会话过期策略
- 依赖关系：依赖 `TODO-2`
- 验收标准：
  - 管理员可成功登录后台
  - 错误账号或密码会返回统一错误
  - 密码以安全摘要存储
  - 会话具备失效时间

### TODO-9：实现后台内容管理

- 目标：支持后台查看内容、查看状态、启用、禁用和逻辑删除
- 主要产出：后台内容接口、状态流转校验、审计日志
- 依赖关系：依赖 `TODO-8`
- 验收标准：
  - 管理员可查看内容列表和详情
  - 可执行启用、禁用、逻辑删除
  - 非法状态流转被正确拦截
  - 所有操作均写入审计日志

### TODO-10：实现手动采集任务入口

- 目标：让后台可以提交手动采集任务，并由调度器近实时消费
- 主要产出：任务创建接口、待执行状态、消费逻辑
- 依赖关系：依赖 `TODO-3`、`TODO-8`
- 验收标准：
  - 后台可创建手动采集任务
  - 任务以 `queued` 状态入库
  - cron 能消费待执行任务
  - 管理员可查看任务结果和失败原因

### TODO-11：实现后台任务与日志查看

- 目标：支持查看任务列表、任务明细、失败摘要和最近异常
- 主要产出：任务查询接口、明细接口、日志页
- 依赖关系：依赖 `TODO-10`
- 验收标准：
  - 可按状态和时间筛选任务
  - 可查看任务成功数、重复数、失败数
  - 可查看逐条处理明细
  - 最近失败事件可快速定位

### TODO-12：实现健康检查与资源巡检

- 目标：提供服务健康检查，并能发现数据库与文件系统不一致的问题
- 主要产出：`live`、`ready`、`deep` 健康检查接口，资源巡检脚本
- 依赖关系：依赖 `TODO-4`、`TODO-7`
- 验收标准：
  - 存活检查可确认进程可响应
  - 就绪检查可验证数据库和关键目录可用
  - 深度检查可返回最近一次采集和磁盘状态摘要
  - 巡检发现资源丢失时可将内容下线或隔离

### TODO-13：实现备份与恢复流程

- 目标：让数据库、资源目录、配置和日志具备可恢复能力
- 主要产出：备份脚本、恢复手册、恢复验证步骤
- 依赖关系：依赖 `TODO-7`
- 验收标准：
  - 可对数据库做一致性备份
  - 可备份图片正式目录和关键配置
  - 恢复后公开接口和后台接口可正常访问
  - 恢复后资源巡检可通过

## 阶段三：增强能力

### TODO-14：增加标签体系

- 目标：支持内容标签和更丰富的筛选能力
- 主要产出：标签模型、标签管理、公开筛选增强
- 依赖关系：依赖 `TODO-5`、`TODO-9`
- 验收标准：
  - 内容可关联多个标签
  - 公开端可按标签筛选
  - 后台可维护标签关系

### TODO-15：扩展多来源采集

- 目标：在 Bing 之外接入更多图片来源
- 主要产出：来源抽象、来源配置、扩展去重策略
- 依赖关系：依赖 `TODO-3`
- 验收标准：
  - 新来源可通过统一采集接口接入
  - 不破坏现有 Bing 主链路
  - 任务和内容记录可区分来源类型

### TODO-16：完善资源派生版本

- 目标：支持缩略图、详情预览图和其他派生资源
- 主要产出：资源版本生成、资源类型区分、前端使用策略
- 依赖关系：依赖 `TODO-4`、`TODO-6`
- 验收标准：
  - 列表页默认使用缩略图而非原图
  - 详情页可区分预览图和下载图
  - 资源表能区分多种资源类型

### TODO-17：适配 OSS 与 CDN

- 目标：让资源存储从本地文件平滑演进到对象存储
- 主要产出：存储后端抽象、资源定位兼容层、迁移策略
- 依赖关系：依赖 `TODO-4`
- 验收标准：
  - 新旧资源可在迁移期并存
  - 公开访问地址不依赖本地磁盘路径
  - 切换存储后公开站点下载能力不受破坏

### TODO-18：增加下载统计与分析

- 目标：记录下载行为并形成基础分析能力
- 主要产出：下载登记、统计报表、热门内容视图
- 依赖关系：依赖 `TODO-5`、`TODO-11`
- 验收标准：
  - 下载动作可被记录
  - 可查看基础下载趋势
  - 不让应用服务承担大文件主传输链路

### TODO-19：增强搜索能力

- 目标：在现有筛选之外提供更完善的搜索能力
- 主要产出：关键词字段、搜索接口、后台搜索能力
- 依赖关系：依赖 `TODO-14`
- 验收标准：
  - 公开端可按关键词查询内容
  - 后台可按关键词和状态联合检索
  - 搜索结果与筛选规则保持一致

## 阶段间依赖摘要

- 阶段二整体依赖阶段一已经打通最小业务闭环
- 阶段三整体依赖阶段一主链路稳定，且部分能力依赖阶段二后台管理和运维能力
- 若阶段一未完成公开 API、资源管理和基础部署，则阶段二和阶段三不应提前展开

## 验收方式建议

- 每完成一条 TODO，都应补充对应实现文档和变更记录
- 每条 TODO 的验收至少包含功能验证、异常验证和回归验证
- 如果某条 TODO 无法完成全部验收，需要记录未验证项、原因和风险
